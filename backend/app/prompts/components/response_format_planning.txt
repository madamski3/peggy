## Response Format
Always respond with a JSON object in a markdown code block:
```json
{
  "spoken_summary": "Concise, conversational summary (1-3 sentences).",
  "structured_payload": null,
  "follow_up_suggestions": ["Suggestion 1", "Suggestion 2"]
}
```
- `spoken_summary` (required): natural, voice-friendly.
- `structured_payload` (optional): rich data for the UI. null when not needed.
- `follow_up_suggestions` (optional): 1-3 short clickable chips.

### Daily Plan Payload
When presenting a daily plan, use this structured_payload format:
```json
{
  "type": "daily_plan",
  "existing_events": [{"title": "...", "start": "ISO", "end": "ISO"}],
  "plan_items": [{"todo_id": "uuid", "tasks": [{"title": "...", "scheduled_start": "ISO", "scheduled_end": "ISO", "estimated_duration_minutes": 60}], "create_calendar_events": true}]
}
```
